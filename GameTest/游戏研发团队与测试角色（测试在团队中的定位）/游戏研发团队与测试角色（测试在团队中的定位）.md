
## 研发团队核心角色（关联测试协作）
### 策划 - 职责：
输出游戏需求文档（PRD）、设计玩法规则 / 数值体系 / 剧情脚本 - 与测试协作：提供需求细节解释、确认需求变更范围、参与缺陷评审（判断是否符合设计预期） - 测试依据：PRD 文档为测试用例编写核心依据，策划设计的数值表 / 剧情分支图为测试验证标准 
### 美术 - 职责：
制作角色 / 场景 / UI 美术资源、输出美术设计规范（配色 / 尺寸 / 风格） - 与测试协作：确认美术资源显示异常（如错位 / 拉伸）、提供资源尺寸 / 格式标准、修复美术相关 bug - 测试关联：测试美术资源在不同分辨率 / 平台下的适配性、动画播放流畅度 
### 程序 - 职责：
实现游戏功能代码、优化性能（帧率 / 内存）、修复功能 bug - 与测试协作：提供功能实现逻辑说明、协助定位复杂 bug（如偶现崩溃）、确认 bug 修复方案 - 测试关联：功能测试验证代码实现是否符合需求、性能测试监控代码运行资源占用 
### 音效 - 职责：
制作背景音乐（BGM）、音效（操作 / 战斗 / 环境音）、语音配音 - 与测试协作：确认音频播放异常（如无声 / 卡顿 / 错位）、提供音频格式 / 音量标准、修复音频相关 bug - 测试关联：测试音频在不同设备（耳机 / 扬声器）下的兼容性、音频与画面同步性 
## 测试团队核心角色 
### 功能测试工程师 - 核心职责：
执行黑盒测试（不关注代码逻辑，验证功能是否符合需求）、部分白盒测试（如接口逻辑验证） - 工作内容：编写功能测试用例、执行用例并记录缺陷、参与需求评审、回归验证修复的 bug - 核心能力：需求理解能力、用例设计覆盖率、缺陷定位准确性 
### 性能测试工程师 - 核心职责：
监控游戏运行时的性能指标，发现性能瓶颈 - 工作内容：测试帧率稳定性（目标：移动平台≥30FPS，主机 / PC≥60FPS）、内存占用（无内存泄漏）、CPU/GPU 使用率、电池功耗（移动平台） - 工具使用：Unity Profiler、Unreal Stat Unit、ADB（Android 性能监控）、Fraps（PC 帧率测试） 
### 兼容性测试工程师 - 核心职责：
验证游戏在不同平台 / 硬件 / 系统版本下的兼容性 - 工作内容：制定兼容性测试矩阵（平台 + 硬件 + 系统版本组合）、执行多设备适配测试、记录设备专属 bug - 核心关注：硬件驱动冲突、系统版本不兼容、设备分辨率适配异常 
### 自动化测试工程师 - 核心职责：
编写自动化测试脚本，提升回归测试效率 - 工作内容：使用工具（如 Appium、Selenium、Unity Test Framework）编写脚本、维护自动化用例库、执行自动化回归测试 - 应用场景：高频重复功能（如登录 / 注册）回归、多版本兼容性批量测试 
### 安全测试工程师 - 核心职责：
检测游戏安全漏洞，防范破解 / 作弊风险 - 工作内容：测试防破解（如加密验证）、防作弊（如外挂检测）、数据安全（用户信息加密）、支付安全 - 测试手段：逆向工程检测、数据包抓包分析、作弊工具模拟测试 
### 本地化测试工程师 - 核心职责：
验证游戏在不同语言 / 区域的适配性与合规性 - 工作内容：测试多语言翻译准确性（无错译 / 漏译）、区域合规（如年龄分级 / 法律条款）、文化适配（如图标 / 文案无文化冲突） - 关注重点：文字排版（无重叠 / 截断）、日期 / 时间 / 货币格式、本地化语音适配 
## 测试团队协作流程 
### 需求评审 - 参与人员：
测试工程师、策划、程序、美术 - 核心目标：确认测试范围（明确需测 / 无需测功能）、识别需求风险（如逻辑矛盾 / 实现难度）、统一需求理解 - 输出产物：需求评审纪要（记录疑问与决议）、初步测试范围清单 
### 用例编写 - 依据文档：
需求文档（PRD）、设计文档（如玩法设计 / 数值设计）、原型图 - 编写标准：用例包含编号、模块、测试点、前置条件、操作步骤、预期结果、优先级（高 / 中 / 低） - 评审与优化：组织用例评审（策划 / 程序参与）、根据评审意见修改用例、形成最终用例库 
### 缺陷跟踪 - 工具选择：
Jira、禅道、TestRail - 缺陷记录标准：包含标题（简洁描述问题）、模块、严重程度（致命 / 严重 / 一般 / 轻微）、优先级、复现步骤、预期结果、实际结果、附件（截图 / 录屏） - 跟踪流程：提交缺陷→开发确认→开发修复→测试回归→关闭 / 重开（未修复） 
### 版本回归 - 回归触发条件：
开发修复 bug 后发布新测试版本、版本迭代更新后 - 回归范围：修复的 bug（验证是否解决）、关联功能（防止修复引入新问题）、核心功能（必回归，如登录 / 支付） - 输出产物：回归测试报告（记录回归结果、未解决 bug 清单）
