### 核心战斗流程测试
#### 战斗启动与退出
- 战斗入口触发：各入口（关卡列表、PVP 匹配、副本传送）点击后能否正常进入战斗场景，加载资源是否完整（角色、地图、怪物模型无缺失）
- 战斗准备阶段：战前配置（选技能、换装备、组队）是否生效，倒计时结束后能否正常开战
- 战斗退出：主动退出（暂停菜单退出）、被动退出（网络中断、闪退、电量不足）后，进度 / 奖励 / 数据是否符合预期（如未完成战斗无奖励、重连后回到战斗场景）
- 战斗胜利 / 失败判定：达成胜利条件（击杀 BOSS、占领据点、存活时间达标）后是否立即结算，失败条件（角色全灭、超时、核心目标被摧毁）触发是否准确
####  战斗内核心交互
- 角色操控：移动（摇杆 / 点击）、普攻 / 技能释放（按键触发、连招衔接）是否响应及时，无延迟 / 卡顿 / 无效操作
- 目标锁定：自动索敌 / 手动选敌是否准确，切换目标是否生效（如从小怪切到 BOSS）
- 交互触发：场景交互（踩机关、捡补给、触发陷阱）是否按规则生效，多人组队时交互权限（如仅队长可开宝箱）是否正确
- 状态同步：多人战斗中（PVP / 组队副本），队友的位置、血量、技能释放效果是否实时同步，无视觉 / 数据错位（如 A 玩家看到 B 玩家已阵亡，B 玩家界面仍显示存活）
### 战斗规则与数值测试
#### 伤害 / 治疗计算

- 基础公式验证：普攻 / 技能伤害 =（攻击力 - 防御力）× 系数 + 额外加成，需代入多组数值验证（如不同等级、装备、buff 下的计算结果是否与策划表一致）
- 暴击 / 闪避 / 格挡：触发概率是否符合配置（如暴击率 20% 需大量测试验证概率偏差），触发后效果（暴击伤害翻倍、闪避无伤害、格挡减伤）是否正确
- 元素 / 属性克制：克制关系（如火克草、物攻克护甲）是否生效（克制时伤害 150%，被克制时 50%），无属性攻击是否无加成 / 减成
- 治疗效果：技能治疗量、回血道具效果是否符合数值表，治疗暴击 / 过量治疗（超出最大血量）是否按规则处理（如过量治疗转化为护盾）

#### 2. 角色 / 怪物状态管理

- 正面状态（Buff）：加速、增伤、护盾、回血等 Buff 的持续时间、效果叠加（如多个加速 Buff 是否叠加数值 / 时长）、清除条件（时效到、被驱散、死亡）是否正确
- 负面状态（Debuff）：中毒、眩晕、冰冻、沉默等 Debuff 的触发条件（如技能命中、踩毒圈）、持续效果（眩晕期间无法操作、中毒持续掉血）、解除方式（吃药、净化技能、死亡复活）是否符合规则
- 死亡与复活：角色死亡后是否无法操作，复活条件（复活道具、队友复活技能、倒计时复活）触发后，血量 / 状态 / 技能 CD 是否重置
- 怪物 AI 行为：怪物的攻击频率、技能释放条件（如血量低于 50% 释放大招）、仇恨目标（优先攻击血量最低 / 输出最高的玩家）是否符合 AI 配置，无逻辑错乱（如怪物原地不动、攻击友方）

#### 3. 资源消耗与产出

- 战斗资源消耗：技能释放消耗蓝量 / 能量、普攻消耗耐力、使用道具消耗数量，是否准确扣除，无无限消耗 / 扣除错误（如释放技能后蓝量未减）
- 战斗奖励产出：胜利后经验、金币、装备、材料的掉落数量 / 概率是否符合配置，失败 / 中途退出是否无奖励，多人组队时奖励分配（按伤害占比 / 平均分配）是否正确
- 资源限制：战斗内补给品（血瓶、蓝瓶）携带数量上限、使用冷却时间是否生效，无超限使用 / 冷却失效

### 技能与装备系统测试

#### 1. 技能模块

- 技能基础属性：技能射程、施法时间、冷却时间、作用范围（圆形 / 矩形 AOE）是否与配置一致（如射程 10 米，超出后无法释放）
- 技能效果：伤害型技能是否命中目标、控制型技能是否限制目标行动、辅助型技能是否给队友加 Buff，无效果缺失（如技能释放后无伤害、控制不生效）
- 技能冲突 / 联动：多个技能同时释放（如连招）是否有冲突（如技能 A 释放中无法触发技能 B），技能联动效果（如技能 A 命中后触发技能 B 额外伤害）是否生效
- 特殊技能：召唤技能（召唤物属性、存活时间、攻击目标）、变身技能（变身后属性加成、形态切换、持续时间）、必杀技（释放条件、动画效果、伤害倍率）是否符合设计

#### 2. 装备 / 道具生效

- 装备属性加成：穿戴 / 卸下装备后，角色攻击力、防御力等属性是否实时更新，无加成不生效 / 叠加错误（如穿两件相同装备只加一次属性）
- 装备特效：装备被动效果（如攻击时有 10% 概率回血、暴击时触发闪电）是否按规则触发，触发条件 / 效果是否准确
- 战斗道具使用：血瓶、蓝瓶、复活币、增益药水等道具，使用后效果是否立即生效，道具数量是否正确扣除，冷却时间是否生效

### 四、性能与兼容性测试

#### 1. 性能测试

- 帧率稳定性：战斗场景（尤其是多人混战、技能特效密集时）帧率是否达标（移动端≥30 帧、PC 端≥60 帧），无掉帧 / 卡顿 / 闪退
- 资源占用：战斗过程中 CPU / 内存 / GPU 占用是否在合理范围，长时间战斗（如 30 分钟副本）是否内存泄漏（占用持续升高）
- 加载性能：战斗场景加载时间是否符合要求（移动端≤5 秒、PC 端≤3 秒），低配设备是否能正常加载（无模型缺失 / 贴图错误）
- 网络性能：多人战斗中，网络延迟（200ms/500ms/1000ms）下是否能正常同步，弱网 / 断网重连后是否恢复战斗状态

#### 2. 兼容性测试

- 设备兼容：不同机型（移动端高低配、PC 不同显卡）、系统版本（iOS 15+/Android 10+/Windows 10/11）下战斗模块是否正常运行，无适配错误（如按钮错位、特效显示异常）
- 分辨率兼容：不同分辨率（1080P/2K/4K、全面屏 / 非全面屏）下战斗界面元素（血条、技能按钮、小地图）是否完整显示，无被遮挡 / 拉伸
- 操作方式兼容：触屏（移动端）、键鼠（PC）、手柄等操作方式下，战斗操控是否流畅，无操作失效（如手柄摇杆无法移动角色）

### 五、边界与异常场景测试

#### 1. 极端数值场景

- 角色属性极值：攻击力 / 防御力拉满 / 为 0 时，伤害计算是否合理（如防御力为 0 时伤害最大化，攻击力为 0 时无法造成伤害）
- 怪物数量极值：单场战斗刷出最大数量怪物（如 100 只小怪）时，是否卡顿 / 崩溃，伤害计算是否正常
- 持续战斗极值：连续进行 10 + 场战斗后，是否出现数据错乱（如血量不恢复、技能 CD 不重置）

#### 2. 异常操作场景

- 高频操作：快速点击技能按钮（每秒 5 次）、频繁切换目标，是否出现技能释放异常（如重复释放、无法释放）
- 无效操作：战斗未开始时释放技能、死亡后点击移动 / 攻击、超出技能射程释放技能，是否有正确提示（如 “战斗未开始”“无法攻击”），无错误触发
- 并发操作：战斗中同时进行多个操作（如移动 + 释放技能 + 使用道具），是否冲突（如操作无响应、数据错误）

#### 3. 系统异常场景

- 网络异常：战斗中断网 / 重连、网络波动（高延迟），重连后战斗进度 / 数据是否同步，无回档 / 奖励错误
- 进程异常：战斗中强制关闭游戏再重启、闪退重启，是否能回到战斗场景（未完成战斗）或正常结算（已完成战斗）
	- 权限异常：战斗中弹出系统权限请求（如相机、定位），关闭后是否回到战斗界面，无卡顿 / 退出
