### 核心战斗流程测试
#### 战斗启动与退出
- 战斗入口触发：各入口（关卡列表、PVP 匹配、副本传送）点击后能否正常进入战斗场景，加载资源是否完整（角色、地图、怪物模型无缺失）
- 战斗准备阶段：战前配置（选技能、换装备、组队）是否生效，倒计时结束后能否正常开战
- 战斗退出：主动退出（暂停菜单退出）、被动退出（网络中断、闪退、电量不足）后，进度 / 奖励 / 数据是否符合预期（如未完成战斗无奖励、重连后回到战斗场景）
- 战斗胜利 / 失败判定：达成胜利条件（击杀 BOSS、占领据点、存活时间达标）后是否立即结算，失败条件（角色全灭、超时、核心目标被摧毁）触发是否准确
####  战斗内核心交互
- 角色操控：移动（摇杆 / 点击）、普攻 / 技能释放（按键触发、连招衔接）是否响应及时，无延迟 / 卡顿 / 无效操作
- 目标锁定：自动索敌 / 手动选敌是否准确，切换目标是否生效（如从小怪切到 BOSS）
- 交互触发：场景交互（踩机关、捡补给、触发陷阱）是否按规则生效，多人组队时交互权限（如仅队长可开宝箱）是否正确
- 状态同步：多人战斗中（PVP / 组队副本），队友的位置、血量、技能释放效果是否实时同步，无视觉 / 数据错位（如 A 玩家看到 B 玩家已阵亡，B 玩家界面仍显示存活）
### 战斗规则与数值测试
#### 伤害 / 治疗计算

- 基础公式验证：普攻 / 技能伤害 =（攻击力 - 防御力）× 系数 + 额外加成，需代入多组数值验证（如不同等级、装备、buff 下的计算结果是否与策划表一致）
- 暴击 / 闪避 / 格挡：触发概率是否符合配置（如暴击率 20% 需大量测试验证概率偏差），触发后效果（暴击伤害翻倍、闪避无伤害、格挡减伤）是否正确
- 元素 / 属性克制：克制关系（如火克草、物攻克护甲）是否生效（克制时伤害 150%，被克制时 50%），无属性攻击是否无加成 / 减成
- 治疗效果：技能治疗量、回血道具效果是否符合数值表，治疗暴击 / 过量治疗（超出最大血量）是否按规则处理（如过量治疗转化为护盾）

#### 2. 角色 / 怪物状态管理

- 正面状态（Buff）：加速、增伤、护盾、回血等 Buff 的持续时间、效果叠加（如多个加速 Buff 是否叠加数值 / 时长）、清除条件（时效到、被驱散、死亡）是否正确
- 负面状态（Debuff）：中毒、眩晕、冰冻、沉默等 Debuff 的触发条件（如技能命中、踩毒圈）、持续效果（眩晕期间无法操作、中毒持续掉血）、解除方式（吃药、净化技能、死亡复活）是否符合规则
- 死亡与复活：角色死亡后是否无法操作，复活条件（复活道具、队友复活技能、倒计时复活）触发后，血量 / 状态 / 技能 CD 是否重置
- 怪物 AI 行为：怪物的攻击频率、技能释放条件（如血量低于 50% 释放大招）、仇恨目标（优先攻击血量最低 / 输出最高的玩家）是否符合 AI 配置，无逻辑错乱（如怪物原地不动、攻击友方）

#### 3. 资源消耗与产出

- 战斗资源消耗：技能释放消耗蓝量 / 能量、普攻消耗耐力、使用道具消耗数量，是否准确扣除，无无限消耗 / 扣除错误（如释放技能后蓝量未减）
- 战斗奖励产出：胜利后经验、金币、装备、材料的掉落数量 / 概率是否符合配置，失败 / 中途退出是否无奖励，多人组队时奖励分配（按伤害占比 / 平均分配）是否正确
- 资源限制：战斗内补给品（血瓶、蓝瓶）携带数量上限、使用冷却时间是否生效，无超限使用 / 冷却失效
